# entity 层结构说明

## 1. 目录定位

`entity` 为数据库实体层，用于映射表结构字段。

## 2. 子目录说明

- `book`：图书实体对象。
- `user`：用户实体对象。
  - 包含 `VerificationDispatchAudit`，用于验证码发送回执审计。

说明：`login` 暂无独立 `entity/login`；验证码回执审计先按 user 注册链路落库，后续可演进为独立审计域。

## 3. 链路涉及

主要在 service 与 mapper 之间传递，不直接作为 controller 入参或出参。

## 4. 约束

- entity 与数据库字段保持一致，避免混入接口展示逻辑。
