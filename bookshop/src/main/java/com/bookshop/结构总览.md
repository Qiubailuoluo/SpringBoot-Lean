# Bookshop 项目结构总览

## 1. 目标

本文件用于说明 `com.bookshop` 下各一级目录职责、核心调用链路与扩展规则，便于新功能接入和长期维护。

## 2. 一级目录职责

- `controller`：接口接入层，处理 HTTP 请求与响应，不写数据库操作。
- `service`：业务编排层，承接业务规则、异常抛出与对象转换。
- `mapper`：数据访问层，负责 SQL 与数据库交互。
- `entity`：数据库实体对象，承载表字段。
- `dto`：接口入参对象，配合参数校验注解。
- `vo`：接口出参对象，统一对外返回数据形态。
- `common`：通用能力层（返回体、trace、错误码枚举等）。
- `exception`：全局异常体系（业务异常 + 全局异常处理器）。
- `config`：系统配置与启动期能力。

## 3. 标准调用链路

`Controller -> Service -> Mapper -> Database`

补充链路：

- 异常链路：`Service throw BusinessException -> GlobalExceptionHandler -> ApiResponse`
- 追踪链路：`TraceIdFilter -> MDC/request -> GlobalExceptionHandler response(traceId)`

## 4. 当前业务域

- `book`：图书模块（已落地 CRUD）
- `user`：用户模块（已落地 CRUD）
- `login`：登录/认证模块（已接入 JWT + Redis：`POST /api/auth/login`、`POST /api/auth/refresh`、`POST /api/auth/logout`、`POST /api/auth/password/change`、`POST /api/auth/password/reset`、`POST /api/auth/verification/send`）
- `system`：系统接口（健康检查）

## 5. 日志与观测（概要）

- 运行时日志由 `classpath:logback-spring.xml` 按 `spring.profiles.active` 切换级别与文件输出；默认滚动目录为工作目录下 `logs/`（已加入仓库 `.gitignore`）。
- 约定与 TraceId、未来 APM 的衔接见 `CursorChat/conventions/日志与观测规范.md`。

## 6. 扩展约定

新增业务域（如 `order`）时，需要同时补齐：

1. `controller/<domain>`
2. `service/<domain>`
3. `mapper/<domain>`
4. `dto/<domain>`
5. `vo/<domain>`
6. `entity/<domain>`
7. `common/enums/<domain>`（如有业务错误码）

说明：`login` 已接入 JWT + Redis，`user_account.password_hash` 使用 BCrypt 校验；当前已支持改密/重置后旧 token 失效。后续安全增强重点转为多端会话治理与认证审计（见 `CursorChat/conventions/演进路线图.md` §5.2）。
