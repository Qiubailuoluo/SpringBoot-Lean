# exception 层结构说明

## 1. 目录定位

`exception` 为全局异常体系层，统一管理业务异常与全局异常收口。

## 2. 子目录说明

当前层级下暂无子目录，核心类包括：

- `BusinessException`：业务异常类型。
- `GlobalExceptionHandler`：全局异常处理器。

## 3. 链路涉及

service 抛出 `BusinessException`，由 `GlobalExceptionHandler` 统一转为 `ApiResponse`。

## 4. 约束

- controller/service 不直接拼接异常返回结构，统一交由全局异常处理器处理。
