# common 层结构说明

## 1. 目录定位

`common` 用于存放跨业务域复用的基础能力，不承载具体业务流程。

## 2. 子目录说明

- `response`：统一返回体对象（`ApiResponse`）。
- `trace`：请求链路追踪能力（`TraceIdFilter`）。
- `enums`：按业务域划分的错误码枚举（如 `book`、`user`）。

## 3. 链路涉及

- 接口响应由 `ApiResponse` 统一包装。
- 每次请求由 `TraceIdFilter` 写入 `traceId`，用于日志与异常响应关联。

## 4. 约束

- `common` 不承载具体业务逻辑，不直接依赖具体业务模块实现。
