# mapper 层结构说明

## 1. 目录定位

`mapper` 为数据访问层，负责与数据库交互。

## 2. 子目录说明

- `book`：图书相关 SQL 访问。
- `user`：用户相关 SQL 访问。

说明：认证持久化（如 refresh_token 表）未落地前不设 `mapper/login`；与 `CursorChat/conventions/演进路线图.md` 中安全章节对齐后再补。

## 3. 链路涉及

由 service 调用 mapper 执行 CRUD，不直接暴露给 controller。

## 4. 约束

- mapper 专注数据访问，不编排业务流程。
