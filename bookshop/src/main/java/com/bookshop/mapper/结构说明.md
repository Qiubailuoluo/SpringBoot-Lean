# mapper 层结构说明

## 1. 目录定位

`mapper` 为数据访问层，负责与数据库交互。

## 2. 子目录说明

- `book`：图书相关 SQL 访问。
- `user`：用户相关 SQL 访问。
  - 包含 `verification_dispatch_audit` 的写入 Mapper，用于验证码发送回执审计落库。

说明：认证主流程仍不设 `mapper/login`；验证码发送回执审计已先落在 `mapper/user`，后续按领域拆分可迁移到独立审计模块。

## 3. 链路涉及

由 service 调用 mapper 执行 CRUD，不直接暴露给 controller。

## 4. 约束

- mapper 专注数据访问，不编排业务流程。
