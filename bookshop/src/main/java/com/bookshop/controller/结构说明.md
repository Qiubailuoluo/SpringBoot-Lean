# controller 层结构说明

## 1. 目录定位

`controller` 为接口接入层，负责接收请求、参数绑定、调用 service、返回统一结果。

## 2. 子目录说明

- `book`：图书模块 HTTP 接口。
- `user`：用户模块 HTTP 接口。
- `login`：认证相关 HTTP 接口（当前为 `/api/auth` 骨架，未启用 Spring Security）。
- `system`：系统级接口（健康检查等）。

## 3. 链路涉及

请求先进入 `controller`，再下发到 `service` 层处理。

## 4. 约束

- 禁止在 controller 中直接写 SQL 或调用 mapper。
- 统一返回 `ApiResponse`。
