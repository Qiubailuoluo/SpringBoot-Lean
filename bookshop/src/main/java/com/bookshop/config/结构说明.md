# config 层结构说明

## 1. 目录定位

`config` 用于放置系统配置与启动期辅助能力。

## 2. 子目录说明

- `startup`：启动阶段执行的校验或初始化逻辑。
- `security`：安全配置（`SecurityFilterChain`、JWT 过滤器、401/403 处理器、JWT 参数绑定）。
- `swagger`：OpenAPI/Swagger3 配置（接口分组、Bearer 鉴权定义）。
- 文档 UI：当前使用 Swagger UI（`/swagger-ui.html`）。
- 配置项扩展：`security.verification.mode` 用于切换验证码发送策略（`mock` / `real`）。

## 3. 链路涉及

应用启动时执行 `startup` 下组件（如数据库连接可用性校验）；请求进入时由 `security` 配置决定鉴权流程。

## 4. 约束

- `config` 层避免承载业务逻辑，仅用于框架配置和系统级能力。
