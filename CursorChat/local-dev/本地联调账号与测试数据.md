# 本地联调账号与测试数据（明文说明）

> 说明：本文件仅用于**本地项目联调**，按你的要求保留明文账号信息，便于快速测试。  
> 但数据库中密码字段仍必须按规范存储为 `BCrypt` 哈希，禁止明文入库。

## 1. 推荐联调账号（明文）

- 用户名：`qiubai`
- 登录密码（联调）：`Abc12345`
- 验证目标：`demo@example.com`
- 显示名：`QiuBai`
- 状态：`1`（启用）

## 2. 数据库存储规范（必须）

- `user_account.password_hash` 仅存哈希值（`BCrypt`），不存明文。
- `user_account.verify_target` 保持唯一（已建 `uk_verify_target`）。
- `user_account.username` 保持唯一（已建 `uk_username`）。
- 所有业务改密/重置必须通过接口走校验链路，避免手工明文写库。

## 3. 初始化建议（接口方式，推荐）

优先通过注册接口初始化账号，让服务自动完成密码哈希：

```http
POST /api/users
Content-Type: application/json

{
  "username": "qiubai",
  "password": "Abc12345",
  "verifyTarget": "demo@example.com",
  "displayName": "QiuBai"
}
```

## 4. SQL 查询示例（只读）

```sql
SELECT id, username, verify_target, password_hash, status, created_at
FROM user_account
WHERE username = 'qiubai';
```

校验点：
- `password_hash` 应为 `$2a$` / `$2b$` 开头的 BCrypt 串；
- `verify_target` 应与账号绑定一致，用于重置密码强校验。
