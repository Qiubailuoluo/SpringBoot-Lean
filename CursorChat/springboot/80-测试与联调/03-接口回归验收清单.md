# 接口回归验收清单

## 目标
每次变更前后有统一回归口径，降低回归遗漏风险。

## 验收清单
- 登录成功、登录失败、账号禁用三类结果正确。
- refresh 可正常换发 access，失效 refresh 返回预期错误码。
- 受保护接口未携带 token 时返回 401。
- 参数校验失败返回 `VALIDATION_400` 且结构完整。
- 业务异常返回模块错误码且带 `traceId`。
- `/health` 可用，数据库连通检查正常。

## 回归执行图
阅读提示：主线是顺序回归，若失败则回流到修复并重新执行闭环。
```mermaid
flowchart TD
startRegression[StartRegression(开始回归)] --> authCases[RunAuthCases(执行认证用例)]
authCases --> coreCrud[RunCoreCrudCases(执行核心CRUD用例)]
coreCrud --> exceptionCases[RunExceptionCases(执行异常路径用例)]
exceptionCases --> healthCheck[RunHealthCheck(执行健康检查)]
healthCheck --> allPass{AllPass(是否全部通过)}
allPass -->|Yes(是)| releaseReady[MarkReleaseReady(标记可发布)]
allPass -->|No(否)| fixAndRetry[FixAndRetry(修复并重跑)]
fixAndRetry --> authCases
```
## 图解摘要
- 回归顺序建议固定为：认证、核心 CRUD、异常路径、健康检查。
- 只有全部通过才允许进入发布准备状态。
- 任意失败都要回流修复并重新执行完整回归链路。

## 对应源码入口
- `java/com/bookshop/controller/login/LoginController.java`
- `java/com/bookshop/controller/book/BookController.java`

## 发布前建议
- 每次发布保留一份回归记录（日期、执行人、结果）。
