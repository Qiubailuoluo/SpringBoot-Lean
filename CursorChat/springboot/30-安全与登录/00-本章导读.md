# 本章导读

## 本章解决什么问题
- Spring Security 在本项目中的真实执行路径。
- JWT + Redis 如何实现无状态认证与会话治理。
- 登录、刷新、登出在失败场景下如何响应。

## 阅读产出
- 能解释 401 与 403 分别从哪里产生。
- 能说明 access/refresh token 的生命周期和失效逻辑。

## 本阶段新增落地点
- 登录/刷新/登出 + 黑名单链路已稳定。
- 改密/重置后旧 token 失效机制已落地。
- 验证码能力支持 `mock/stub` 策略与发送回执统一结构。
- `username + verifyTarget` 强绑定已补齐，收口重置密码越权窗口。
- refresh token 已支持 `deviceId` 维度的多端会话增强。
