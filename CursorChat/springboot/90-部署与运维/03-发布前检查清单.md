# 发布前检查清单

## 目标
在上线前做最小但关键的质量闸门检查。

## 检查项
- 环境变量与 profile 是否正确。
- 数据库变更脚本已在测试环境验证。
- Redis 与数据库连接正常。
- 登录链路与核心业务接口回归通过。
- 错误码、日志、traceId 输出符合规范。
- 回滚方案与责任人已明确。

## 发布闸门图
阅读提示：各检查项依次通过后才放行发布，任一失败都应阻断并修复。
```mermaid
flowchart LR
configCheck[ConfigCheck] --> dbCheck[DbMigrationCheck]
dbCheck --> redisCheck[RedisCheck]
redisCheck --> regressionCheck[RegressionCheck]
regressionCheck --> observabilityCheck[LogsAndTraceCheck]
observabilityCheck --> rollbackReady[RollbackPlanReady]
rollbackReady --> gateResult{GatePass}
gateResult -->|Yes| releaseExec[ExecuteRelease]
gateResult -->|No| blockRelease[BlockAndFix]
```
## 图解摘要
- 发布前检查是串行闸门，前一项不过不应进入下一项。
- 配置、数据、回归、观测、回滚准备缺一不可。
- 仅当所有闸门通过，才允许执行正式发布。

## 对应源码入口
- `bookshop/src/main/resources/application-prod.yml`
- `bookshop/src/main/resources/logback-spring.xml`
