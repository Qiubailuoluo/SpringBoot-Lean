# 本地可观测一键自检脚本

## 目标
在联调前 10 秒内确认本地观测链路是否可用，避免“服务没起来就开始排障”。

## 脚本位置
- `CursorChat/运维手册/本地可观测一键自检.ps1`

## 检查项
- Elasticsearch HTTP（默认 `http://127.0.0.1:9200`）
- Kibana 状态 API（默认 `http://127.0.0.1:5601/api/status`）
- SpringBoot 健康接口（默认 `http://127.0.0.1:8080/health`）
- Redis TCP（默认 `127.0.0.1:6379`）
- Filebeat 进程是否存在（默认进程名 `filebeat`）

## 使用方式
在项目根目录执行：
```powershell
powershell -ExecutionPolicy Bypass -File ".\CursorChat\运维手册\本地可观测一键自检.ps1"
```

## 参数覆盖示例
如果端口不同，可传参覆盖：
```powershell
powershell -ExecutionPolicy Bypass -File ".\CursorChat\运维手册\本地可观测一键自检.ps1" `
  -ElasticsearchUrl "http://127.0.0.1:19200" `
  -KibanaStatusUrl "http://127.0.0.1:15601/api/status" `
  -AppHealthUrl "http://127.0.0.1:18080/health" `
  -RedisHost "127.0.0.1" -RedisPort 16379
```

## 输出说明
- 全部通过：退出码 `0`，可继续联调。
- 有失败项：退出码 `1`，先修复基础依赖再继续。
