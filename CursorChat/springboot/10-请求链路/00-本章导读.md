# 本章导读

## 本章解决什么问题
- 一次 HTTP 请求进入系统后，到底经过了哪些组件。
- Controller、Service、Mapper 在链路中的真实职责分工。
- 统一响应和 traceId 为什么能提升排障效率。

## 推荐阅读顺序
1. `01-HTTP请求到Controller链路总览.md`
2. `02-Controller入参与校验机制.md`
3. `03-Service编排与事务边界.md`
4. `04-Mapper与SQL执行路径.md`
5. `05-统一响应与TraceId流转.md`

## 阅读产出
- 能口述请求主链路：Tomcat -> Filter -> Security -> MVC -> Controller -> Service -> Mapper -> DB。
- 能判断一个问题属于入口层、业务层还是数据层。
