# 初始化脚本与迁移策略

## 目标
说明当前项目如何管理初始化 SQL 和表结构变更。

## 脚本位置
- `bookshop/src/main/resources/sql/book_info.sql`
- `bookshop/src/main/resources/sql/user_account.sql`
- `bookshop/src/main/resources/sql/user_account_password_migration.sql`

## 当前策略
- 使用手工 SQL 脚本管理初始化和补丁迁移。
- 目前未接入 Flyway/Liquibase。
- 迁移脚本应保持幂等或明确执行前置条件。

## 迁移执行流程图
阅读提示：从左到右看，先测试库演练与校验，再准备回滚，最后进入生产执行窗口。
```mermaid
flowchart LR
newSchemaNeed[NewSchemaChange] --> writeSql[WriteMigrationSql]
writeSql --> testDbRun[RunInTestDb]
testDbRun --> verifyData[VerifyDataAndApi]
verifyData --> prepareRollback[PrepareRollbackSql]
prepareRollback --> prodExecute[ExecuteInProdWindow]
prodExecute --> postCheck[PostDeployCheck]
```
## 图解摘要
- 结构变更先写脚本，再在测试库完整演练。
- 通过校验后必须先准备回滚脚本，再进入生产窗口执行。
- 生产执行后还要做发布后检查，形成闭环。

## 对应源码入口
- `bookshop/src/main/resources/sql/user_account_password_migration.sql`
- `bookshop/src/main/resources/sql/user_account.sql`

## 执行建议
- 新迁移脚本命名带日期与目标：`yyyyMMdd-xxx.sql`。
- 上线前先在测试库演练，并保留回滚 SQL。
