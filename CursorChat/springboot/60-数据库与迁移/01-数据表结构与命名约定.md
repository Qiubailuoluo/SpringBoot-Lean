# 数据表结构与命名约定

## 目标
统一当前项目表结构命名和 Java 字段映射规则。

## 代码与脚本位置
- `resources/db/migration/V1__create_book_info.sql`
- `resources/db/migration/V2__create_user_account.sql`
- `resources/db/migration/V3__create_verification_dispatch_audit.sql`
- `resources/db/migration/V5__add_user_verify_target.sql`
- `resources/application.yml`

## 现状约定
- 表名使用下划线：如 `book_info`、`user_account`。
- 审计表已落地：`verification_dispatch_audit`（验证码发送回执审计）。
- Java 字段驼峰映射依赖 MyBatis 配置：`map-underscore-to-camel-case: true`。
- 密码字段使用 `password_hash`，禁止明文字段。
- 用户表已补充 `verify_target`，用于重置密码绑定校验。

## 建议
- 时间字段统一 `created_at/updated_at`。
- 索引命名建议前缀化：`idx_`（普通索引）、`uk_`（唯一索引）。
