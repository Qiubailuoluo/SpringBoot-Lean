# 本地可观测启动顺序（Redis + SpringBoot + ES + Filebeat + Kibana）

> 目标：确保“接口可调用 + 日志可采集 + Kibana 可查询”稳定可用。

## 1. 推荐启动顺序（按依赖）

1. `Redis`
2. `Elasticsearch`
3. `Kibana`
4. `SpringBoot`
5. `Filebeat`

说明：
- SpringBoot 登录链路依赖 Redis（JWT/限流等）。
- Kibana 依赖 Elasticsearch。
- Filebeat 最后启动，确保 ES 已就绪且应用日志文件已产生。

---

## 2. 每一步验证方式

### 2.1 Redis

- 验证端口：`6379` 可达
- 若你本机有 `redis-cli`：
  - `redis-cli ping` 预期返回 `PONG`

### 2.2 Elasticsearch

- 验证：
  - `Invoke-WebRequest http://127.0.0.1:9200`
- 预期：返回版本与集群 JSON（如 `9.1.2`、`bookshop-local`）

### 2.3 Kibana

- 验证：
  - 浏览器打开 `http://127.0.0.1:5601`
  - 或 `Invoke-WebRequest http://127.0.0.1:5601/api/status`
- 预期：`overall.level = available`

### 2.4 SpringBoot

- 验证：
  - `GET /health`
- 预期：`{"service":"bookshop","status":"UP"}`

### 2.5 Filebeat

- 启动前：
  - `.\filebeat.exe test config -c .\filebeat.yml -e`
  - `.\filebeat.exe test output -c .\filebeat.yml -e`
- 启动后：
  - `.\filebeat.exe -e -c .\filebeat.yml`
- 预期：无配置错误，且持续有 events 输出

---

## 3. Kibana 查询验证（最终验收）

1. 创建 Data View：`bookshop-logs*`
2. 打开 Discover，时间改到 `过去15分钟` 或 `过去1小时`
3. 看到日志即打通

注意：
- 如果你手动建过空索引 `bookshop-logs`，建议删除，避免干扰查询。
- 实际日志常写入数据流索引（`.ds-bookshop-logs-*`）。

---

## 4. 停止顺序（建议）

1. `Filebeat`
2. `SpringBoot`
3. `Kibana`
4. `Elasticsearch`
5. `Redis`

---

## 5. 常见问题

- Kibana 打不开：先检查 ES 是否可访问（9200）
- Discover 无数据：检查 Data View、时间范围、日志路径、Filebeat 配置缩进
- 登录接口 500：优先检查 Redis 是否启动
- 有索引无文档：检查是否写入了数据流索引（`.ds-...`）
