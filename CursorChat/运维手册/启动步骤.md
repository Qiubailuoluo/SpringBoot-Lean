# 本地启动步骤（可观测链路）

适用场景：`Redis + SpringBoot + Elasticsearch + Kibana + Filebeat` 本地联调。

## 1. 推荐启动顺序

1. 启动 Redis（认证链路依赖）
2. 启动 Elasticsearch（日志存储）
3. 启动 Kibana（日志 UI）
4. 启动 SpringBoot（产生业务日志）
5. 启动 Filebeat（采集并发送日志）

---

## 2. 各组件启动与验证

### 2.1 Redis

- 启动方式：按你本机安装方式启动 Redis 服务或 `redis-server`
- 验证（可选）：

```powershell
redis-cli ping
```

预期返回：`PONG`

### 2.2 Elasticsearch

进入目录（示例）`D:\tools\elastic\elasticsearch-9.1.2`，执行：

```powershell
.\bin\elasticsearch.bat
```

验证：

```powershell
Invoke-WebRequest http://127.0.0.1:9200
```

返回 JSON 即成功。

### 2.3 Kibana

进入目录（示例）`D:\tools\elastic\kibana-9.1.2`，执行：

```powershell
.\bin\kibana.bat
```

验证：

```powershell
Invoke-WebRequest http://127.0.0.1:5601/api/status
```

`overall.level = available` 即成功。

### 2.4 SpringBoot

在项目目录 `bookshop` 启动应用（示例）：

```powershell
mvn spring-boot:run
```

验证：

```powershell
Invoke-WebRequest http://127.0.0.1:8080/health
```

返回 `status=UP` 即成功。

### 2.5 Filebeat

进入目录（示例）`D:\tools\elastic\filebeat-oss-9.1.2`。

先做配置和输出测试：

```powershell
.\filebeat.exe test config -c .\filebeat.yml -e
.\filebeat.exe test output -c .\filebeat.yml -e
```

再启动（前台调试）：

```powershell
.\filebeat.exe -e -c .\filebeat.yml
```

---

## 3. Kibana 最终验收

1. 打开 `http://127.0.0.1:5601`
2. Data View 使用 `bookshop-logs*`
3. Discover 时间范围改为 `过去15分钟` 或 `过去1小时`
4. 调几次接口后刷新，能看到新日志即打通

---

## 4. 停止顺序（建议）

1. Filebeat
2. SpringBoot
3. Kibana
4. Elasticsearch
5. Redis

---

## 5. 常见问题

- Kibana 打不开：先检查 ES `9200` 是否可用
- Discover 无数据：检查时间范围、Data View、Filebeat 路径与缩进
- 接口报 `SYSTEM_500` 且认证链路异常：优先检查 Redis
- 建了索引但没文档：检查是否写入了数据流索引（`.ds-bookshop-logs-*`）
