# MySQL 连接参数约定

## 1. 开发环境默认参数

- 主机：`127.0.0.1`
- 端口：`3306`
- 数据库：`bookshop`
- 用户名：`bookshop_dev`
- 时区：`Asia/Shanghai`
- 字符集：`utf8mb4`
- 排序规则：
  - 首选 `utf8mb4_0900_ai_ci`
  - 兼容 `utf8mb4_general_ci`

## 1.1 当前数据库实况（已确认）

- 数据库：`bookshop`
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_0900_ai_ci`
- 备注：该库已手动创建完成，可进入应用连接阶段

## 2. JDBC URL 模板

```text
jdbc:mysql://127.0.0.1:3306/bookshop?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
```

## 3. Spring Boot 示例配置片段

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/bookshop?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: bookshop_dev
    password: ChangeMe_123456
```

## 4. 安全与协作约定

- 禁止将真实生产密码写入仓库。
- 本地开发可使用占位密码，真实密码通过环境变量注入。
- 团队协作时，统一库名、端口和时区，减少“我这边可以你那边不行”的环境差异。

## 5. 常见问题排查

- 连接失败（拒绝连接）：确认 MySQL 服务已启动，端口是否为 `3306`。
- 认证失败（Access denied）：确认账号授权主机是 `localhost` 或 `%`，并检查密码。
- 时区报错：确认 URL 中 `serverTimezone=Asia/Shanghai` 已配置。
- 字符乱码：确认库、表、连接参数均使用 `utf8mb4`。
