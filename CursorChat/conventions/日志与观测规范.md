# 日志与观测规范

构建与本地终端请先使用 **JDK 17** 并正确设置 `JAVA_HOME`（本机路径见 `CursorChat/conventions/环境与命名规范.md` §4.1，PowerShell 临时切换示例见 `CursorChat/springboot/01-项目初始化参数清单.md` §6.2）。

## 1. Logback 配置位置

- 主配置：`bookshop/src/main/resources/logback-spring.xml`
- 使用 `logback-spring.xml`（而非 `logback.xml`）以便按 Spring `spring.profiles.active` 切换 appender 与级别，并与 `application-*.yml` 的环境模型一致。

## 2. 环境行为摘要

| Profile | 控制台 | 滚动文件 | `com.bookshop` 级别 | 说明 |
|--------|--------|----------|---------------------|------|
| `dev` | 是 | `logs/bookshop-dev*.log` | `DEBUG` | 开发排障，文件保留约 14 天 |
| `test` / `prod` | 是 | `logs/bookshop*.log` | `INFO` | 避免生产 `DEBUG` 刷屏；保留约 30 天 |
| 未匹配上述 profile | 仅控制台 | 无 | `INFO`（仅 root） | 兜底 |

日志目录默认 `logs/`（相对应用工作目录），可通过 JVM 或环境变量覆盖 `LOG_DIR`。

## 3. TraceId 与 MDC

- `TraceIdFilter` 将链路 ID 放入 MDC，键名为 **`traceId`**（与 `com.bookshop.common.trace.TraceIdFilter.TRACE_ID_KEY` 一致）。
- 日志 Pattern 中使用 **`%X{traceId:-}`**（或等价写法），无请求上下文时输出为空，避免占位符报错。
- 响应头 `X-Trace-Id` 与异常体中的 `traceId` 便于前后端对齐；日志检索时优先用该值。

## 4. 与 SkyWalking 的关系（规划）

接入 SkyWalking 后，进程内会存在 APM 维度的 Trace/Segment ID，与上述 MDC `traceId` **不一定相同**。演进策略见 `CursorChat/conventions/演进路线图.md` 第 5.1 节：在文档与排障流程中约定「以哪条链路为主、如何互相对照」。

## 5. 实践建议

- 业务代码使用 SLF4J（`LoggerFactory.getLogger`），避免直接依赖 Logback API。
- 不在日志中输出密码、Token、完整证件号等敏感字段。
- 异步场景若需保留 `traceId`，需在任务提交前传递 MDC（见演进路线图 5.3）。
