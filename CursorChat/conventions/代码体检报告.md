# 代码体检报告（2026-04）

本报告对应“代码体检与架构对标”阶段执行结果，聚焦后端代码质量、认证链路风险与可维护性，不包含部署层。

## 1. 检查范围

- 认证与用户主链路：`login` / `user` / `security` / `exception` / `mapper`
- 测试与文档一致性：`src/test`、`CursorChat` 下规范与联调文档

## 2. 主要发现（按优先级）

### P0（需优先收敛）

1. **验证码明文返回风险**
   - 现状：验证码发送接口会返回 `code` 字段，存在误用于生产的风险。
   - 处理：已新增配置 `security.verification.expose-code`，并在 `prod` 关闭。
   - 变更文件：
     - `bookshop/src/main/java/com/bookshop/controller/login/LoginController.java`
     - `bookshop/src/main/resources/application.yml`
     - `bookshop/src/main/resources/application-prod.yml`

2. **重置密码链路“账号-验证码目标绑定”已补齐（本轮修复）**
   - 现状：`user_account` 新增 `verify_target` 字段并加唯一索引；重置密码时强校验 `username` 与 `verifyTarget` 绑定关系。
   - 结论：该越权窗口已收口；后续可再升级为 `email/mobile` 双字段模型。

### P1（建议本阶段后续推进）

1. **授权粒度仍偏粗**
   - 现状：除白名单外，多数接口是“登录即访问”，未细分角色权限。
   - 建议：引入角色声明（如 `ROLE_ADMIN`）与方法级授权注解。

2. **错误码与 HTTP 语义需要进一步统一**
   - 现状：已有统一 `ApiResponse` 与错误码，但仍需持续检查业务异常状态码映射是否稳定。
   - 建议：在测试中增加错误码+HTTP 状态断言组合。
